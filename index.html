AI is powerful, but unsafe by default.

Unfiltered AI requests can leak PII, trigger prompt injections, and create compliance risk. Straja sits between your app and any model, inspecting every request and response in real time.

What Straja does

  • PII & secrets redaction

    Mask or remove sensitive data in both requests and responses.

  • Injection & jailbreak detection

    Block or route suspicious prompts and tool calls.

  • Policy enforcement

    Allow, deny, or transform requests and responses with simple rules and guardrails.

  • Full audit trail

    Structured logs without exposing raw user data.

Example: gateway policy

/policies/llm.json 
{
  "name": "default_safety",
  "match": { "path": "/v1/chat/completions" },
  "filters": [
    { "type": "pii_redact", "mode": "mask", "scope": "request,response", "entities": ["email","phone","iban"] },
    { "type": "prompt_injection", "action": "block" },
    { "type": "jailbreak", "action": "review" }
  ],
  "limits": { "tokens": 4096, "rate_per_min": 60 },
  "on_violation": { "action": "transform", "template": "Your request was adjusted for safety." },
  "activate": [
    { "sink": "stdout" },
    { "sink": "http", "url": "https://siem.local/ingest/straja", "mode": "stream" }
  ]
}

FAQ

How is Straja different from a general API gateway?

It’s purpose-built for AI safety and observability. Policies and classifiers operate on prompts, responses, and embeddings; not just HTTP/JSON.

Can I run fully offline?

Yes. The gateway and policy engine run locally. Classifier bundles are signed and can be updated via offline mirrors.

Which models are supported?

Any HTTP-addressable model: OpenAI, Anthropic, Azure, Vertex, or self-hosted ones like Ollama and vLLM.

Does Straja see or store my data?

No. Straja operates entirely within your infrastructure and never transmits or stores your model traffic externally.

Can I define my own policies?

Yes. You can define policies in JSON or YAML to control filtering, thresholds, and actions. Straja enforces them locally at runtime.

Is Straja compatible with Kubernetes or Docker?

Yes. Straja can run as a containerized gateway, sidecar, or inline proxy inside Kubernetes, Docker Compose, or edge environments.

Does Straja help with EU AI Act compliance?

Straja provides the technical foundation for EU AI Act readiness by keeping inference data local, enforcing transparent policies, and generating auditable logs for every model interaction. It helps teams meet traceability and risk management requirements without external data exposure.
