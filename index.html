AI is powerful — but unsafe by default.

Unfiltered AI requests can leak PII, trigger prompt injections, and create compliance risk. Straja sits between your app and any model, inspecting every request and response in real time.

  • • PII & secrets redaction
  • • Injection & jailbreak detection
  • • Policy enforcement (allow/deny/transform)
  • • Full audit trail — without exfiltrating raw data

Example policy (JSON)

/policies/llm.json 
{
  "name": "default_safety",
  "match": { "path": "/v1/chat/completions" },
  "filters": [
    { "type": "pii_redact", "mode": "mask", "entities": ["email","phone","iban"] },
    { "type": "prompt_injection", "action": "block" },
    { "type": "jailbreak", "action": "review" }
  ],
  "limits": { "tokens": 4096, "rate_per_min": 60 },
  "on_violation": { "action": "transform", "template": "Your request was adjusted for safety." },
  "activate": [
    { "sink": "stdout" },
    { "sink": "http", "url": "https://siem.local/ingest/straja" }
  ]
}

FAQ

How is Straja different from a general API gateway?

It’s purpose-built for AI safety and observability. Policies and classifiers operate on prompts, responses, and embeddings, not just HTTP/JSON.

Can I run fully offline?

Yes. The gateway and policy engine run locally. Classifier bundles are signed and can be updated via offline mirrors.

Which models are supported?

Any HTTP-addressable model (OpenAI, Anthropic, Azure, Vertex, self-hosted via Ollama, vLLM, etc.).
